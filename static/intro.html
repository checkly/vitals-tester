Core Web Vitals-Tester

not a developer?

This is a small utility to generate websites with specific web vital values. If you want to learn more about web vitals check out web.dev/vitals

If you're developing something that measures vitals, this should be pretty helpful as you can make sure you are doing it right :-) by generating any value you are interested in.

General

Every metric supports categories (GOOD, POOR, NEEDS_IMPROVEMENT) which will randomly pick a value to match that category. Or numerical values with optional random parameters.

F.ex. ?lcp=1000 sets the lcp to 1000, ?lcp=1000&random=500 sets the lcp to some value between 500 and 1500. lcp?=GOOD sets it to a randomly picked "GOOD" value for the metric.

type={DESKTOP|MOBILE} set mobile or desktop thresholds when using categorical values (GOOD, POOR, NEEDS_IMPROVEMENT) random=Number set a random fluctuation of +/-Number

Non-Developers

Hey, I am not a dev! What does all of this vital stuff mean?

Core Web Vitals measure essential aspects of user experience. I created some nice example sites that demo what bad metric values mean.
  • Total Blocking Time - this site has a lot of lag, try to click links and hover over them to understand why high blocking times are bad.
  • First/Largest Contentful Paint LCP and FCP make sure the first and the largest element with content are visible quickly so you can see your content.
  • Cumulative Layout Shift CLS makes sure the layout of a page does not "hop" around too much.
